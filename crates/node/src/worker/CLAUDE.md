# Worker Module

Worker binary scaffold for the Graphene compute network.

## Structure

```
worker/
├── mod.rs      # Worker struct, stats, re-exports
├── config.rs   # WorkerConfig TOML parsing
├── daemon.rs   # run_daemon, signal handling, event loops
├── error.rs    # WorkerError enum
└── solana.rs   # SolanaClient for on-chain operations
```

## Key Types

- **Worker**: Main worker struct with P2P node, stats, lifecycle management
- **WorkerConfig**: Top-level TOML configuration with nested sections
- **WorkerError**: Unified error type for all worker operations
- **SolanaClient**: Raw Solana SDK client for register/unregister/status

## CLI Commands

Binary: `graphene-worker`

| Command | Description |
|---------|-------------|
| `run` | Start the worker daemon |
| `register` | Register on-chain with stake |
| `unregister` | Unregister and reclaim stake |
| `status` | Show worker status |
| `version` | Show version |

## Configuration

See `worker.toml.example` in the repository root.

Sections:
- `[worker]` - name, capabilities, pricing
- `[p2p]` - storage path, relay, port
- `[solana]` - RPC URL, keypair, program ID
- `[vmm]` - Firecracker settings
- `[logging]` - level, format

## Daemon Flow

1. Load config, initialize P2P via `GrapheneNode::new()`
2. Subscribe to `TopicId::compute_v1()` gossip topic
3. Broadcast initial `WorkerAnnouncement`
4. Spawn heartbeat loop (30s interval)
5. Main select loop: gossip events vs shutdown signal
6. Graceful shutdown on SIGINT/SIGTERM

## Integration Points

- Uses `P2PNetwork` trait from `p2p/mod.rs`
- Uses `GrapheneNode` implementation from `p2p/graphene.rs`
- Uses message types from `p2p/messages.rs`
- Uses `TopicId` from `p2p/types.rs`

## Solana Integration

Uses raw `solana-sdk` instructions (not anchor-client) for simplicity.
PDA derivation: `[b"worker", authority.key()]`

Instruction discriminators computed via SHA256 of `global:{instruction_name}`.
